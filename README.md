# Traffic Disaster Resilience Simulation
**Stress‚ÄëTesting Bangladesh‚Äôs Transport Network with Agent‚ÄëBased Simulation for Disaster Readiness**

This repository provides an end‚Äëto‚Äëend workflow to evaluate **transport network resilience** under **disaster scenarios** using an **agent‚Äëbased model (ABM)** coupled with network analytics. It simulates how disruptions (e.g., flooding, blocked links, capacity loss) ripple through origin‚Äëdestination demand, route choice, and system performance, producing **KPIs for readiness and recovery**.

> Goal: give decision‚Äëmakers a transparent, scenario‚Äëdriven tool to test vulnerabilities, compare mitigation options, and communicate trade‚Äëoffs.

---

## ‚ú® Key Features
- **Agent‚ÄëBased Simulation** of travelers/vehicles reacting to disrupted networks.
- **Hazard Scenarios**: parameterize link/node failures, capacity degradation, or dynamic closures.
- **Network Analytics**: betweenness/criticality, disconnected demand, travel‚Äëtime inflation.
- **Policy Levers**: add hardening, re‚Äërouting, staging, or prioritized lanes; measure effects.
- **Reproducible Runs**: configuration‚Äëdriven experiments; cached results and plots.
- **Clear Outputs**: CSVs and figures for KPIs and a compact run log.

## üß© How It Works (Conceptual)

1. **Load network & demand** ‚Äî Read a road/rail graph (edges with length, free‚Äëflow speed/capacity) and OD demand (agents or flows).
2. **Apply disaster scenario** ‚Äî Mark edges/nodes with failure states or capacity multipliers (0‚Äì100%). Optionally time‚Äëvarying.
3. **Simulate agent decisions** ‚Äî Agents pick routes based on current network cost (e.g., travel time), re‚Äërouting when links fail.
4. **Measure system KPIs** ‚Äî Travel‚Äëtime inflation, unmet demand, detour ratios; criticality metrics; recovery profiles if the scenario evolves.
5. **Compare policies** ‚Äî Re‚Äërun with mitigation (hardening, traffic management), quantify benefits vs. baseline.
